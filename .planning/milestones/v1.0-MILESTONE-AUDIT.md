## Integration Findings

### Connected Wiring (22 cross-phase connections verified)
All major cross-phase data flows are complete:
- `lib/types.ts` OgData → 15+ consumer files
- `lib/messaging.ts` → background.ts + content.tsx (both message handlers registered)
- `lib/og-parser.ts` → background.ts + content.tsx (parse + extract)
- `lib/cache.ts` → background.ts (fetch pipeline)
- `lib/og-display.ts` → 8 platform card components + App.tsx + MetadataTab
- `lib/settings.ts` → App.tsx + main.tsx + SettingsPage + content.tsx
- `content.tsx` → TooltipApp → OgTooltip (shadow DOM fully wired)

### Integration Issues

**1. host_permissions vs optional_host_permissions (CORE-03)**
- **Severity:** Critical tech debt
- `wxt.config.ts:20` uses `host_permissions: ['<all_urls>']`
- Production manifest confirms `host_permissions` only — no `optional_host_permissions` key
- Phase 01 VERIFICATION.md falsely claimed this was verified with optional_host_permissions
- **Impact:** CWS submission will show broad "Read and change all your data on all websites" install warning; may trigger manual review
- **Fix:** Change `wxt.config.ts` to use `optionalPermissions` or `optional_host_permissions` + add `chrome.permissions.request()` flow at first use (or accept the blanket permission with CWS justification)

**2. ThemeProvider system-preference listener ignores stored theme (CORE-05)**
- **Severity:** Minor (corner case)
- `components/theme-provider.tsx:7`: `document.documentElement.classList.toggle('dark', e.matches)` fires on OS theme change without checking stored `theme` setting
- If user has set explicit 'dark' in settings and OS switches to light mode, popup will revert to light on next system-preference change
- Phase 08 workaround (SettingsPage directly toggles `.dark` class) works for the active settings session but not for background `prefers-color-scheme` listener
- **Fix:** `ThemeProvider` should check `getSettings().then(s => { if (s.theme === 'system') classList.toggle('dark', e.matches) })` before responding to media query changes

**3. Dead code**
- `getEffectiveTitle`, `getEffectiveDescription`, `getEffectiveImage` — exported from `lib/og-parser.ts` but never imported. `resolveDisplayData` is used instead.
- `components/ui/empty.tsx` — shadcn Empty component family exported but never imported. `EmptyState.tsx` is the actual implementation.

## Tech Debt by Phase

### CRITICAL: Phase 01 — optional_host_permissions not implemented
- Original decision: `[Research]: optional_host_permissions from day one to avoid CWS rejection`
- Actual implementation: `host_permissions: ['<all_urls>']` in wxt.config.ts
- Phase 01 VERIFICATION.md contains a **false positive** — it verified a property that doesn't exist in the manifest
- **Action needed before CWS submission:** Either implement `optional_host_permissions` properly or document the deliberate choice to use blanket permissions with CWS justification

### Phase 03 — ThemeProvider corner case
- `components/theme-provider.tsx:7` overrides explicit user theme setting when OS preference changes
- Minor UX bug; dark mode functionally works in the common case
- **Action:** Add stored-theme check in the `change` event handler

- **Action:** Remove or unexport
