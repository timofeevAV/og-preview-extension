# Requirements: OG Preview Extension

**Defined:** 2026-02-18
**Core Value:** See exactly how any page looks when shared on any platform, right now, without posting it.

## v1 Requirements

### Popup -- Mini Mode

- [ ] **POPUP-01**: User can click the toolbar icon to see a compact OG preview card for the current page
- [ ] **POPUP-02**: User can expand the compact popup to open the full detailed view

### Popup -- Platform Previews

- [ ] **PLATF-01**: User can see a pixel-accurate X/Twitter card preview for the current page
- [ ] **PLATF-02**: User can see a pixel-accurate Facebook desktop card preview for the current page
- [ ] **PLATF-03**: User can see a pixel-accurate Facebook mobile card preview for the current page
- [ ] **PLATF-04**: User can see a pixel-accurate LinkedIn card preview for the current page
- [ ] **PLATF-05**: User can see a pixel-accurate iMessage rich link preview for the current page
- [ ] **PLATF-06**: User can see a pixel-accurate WhatsApp link card preview for the current page

### Popup -- Metadata & Export

- [ ] **META-01**: User can view a raw metadata tab showing all og:* and twitter:* meta tag properties and their values
- [ ] **META-02**: User can see which OG fields are missing, with a per-field explanation of what each field does
- [ ] **META-03**: User can copy all OG metadata as JSON to clipboard with one click
- [ ] **META-04**: User can download OG metadata as a .json file
- [ ] **META-05**: User can copy ready-to-paste HTML `<meta>` tag snippets for missing or present fields

### Hover Tooltip

- [ ] **TTIP-01**: User sees an OG preview tooltip when hovering over any link on a page (fetch triggered after 300ms hover delay)
- [ ] **TTIP-02**: Tooltip dismisses when user moves the mouse away from the link
- [ ] **TTIP-03**: Tooltip shows a loading state while OG data is being fetched
- [ ] **TTIP-04**: Tooltip shows a clear empty/error state when the linked page has no OG tags or fetch fails

### Core & Compatibility

- [ ] **CORE-01**: Extension parses og:* and twitter:* meta tags from the current page's DOM
- [ ] **CORE-02**: Extension shows a clear empty state when the current page has no OG tags, indicating which fields are missing
- [ ] **CORE-03**: Extension works on Chrome, Edge, and Brave (all Chromium via Manifest V3)
- [ ] **CORE-04**: Extension works on localhost and local development environments
- [ ] **CORE-05**: Extension supports dark mode (designed from day one, not retrofitted)

## v2 Requirements

### Platform Enhancements

- **PLATF-07**: Image dimension and aspect ratio validation per platform with actionable warnings
- **PLATF-08**: Per-platform validation warnings (e.g. "image below 1200x630 minimum for Facebook")

### Developer Tools

- **DEV-01**: Links to official platform debugger tools (Facebook Sharing Debugger, Twitter Card Validator, LinkedIn Post Inspector)
- **DEV-02**: Structured data / JSON-LD display alongside OG metadata

### UX

- **UX-01**: Chrome Side Panel mode (persistent docked panel alongside the page)
- **UX-02**: Keyboard navigation and full ARIA accessibility across all popup views

## Out of Scope

| Feature | Reason |
|---------|--------|
| OG tag editor / generator | Misleading -- social crawlers fetch from the server, not the DOM; editing DOM tags won't change how platforms see the page |
| Server-side re-fetch proxy | Requires infrastructure; privacy concerns; official platform debugger links cover this use case |
| Browsing history of checked pages | Storage bloat, privacy implications, low-frequency use case |
| Automatic scanning of all links on page load | Performance disaster on link-dense pages; fetch-on-hover is intentional |
| Firefox support | Single Chromium MV3 codebase; Firefox is out of scope |
