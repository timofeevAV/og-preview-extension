  <how-to-verify>

**Test 1 — Loading state (TTIP-03):**
Visit a page with external links (e.g., https://news.ycombinator.com).
Hover slowly over any external link (hold mouse still for 300ms).
Confirm: A tooltip appears showing a skeleton/shimmer (grey placeholder blocks).

**Test 2 — Ready state (TTIP-01):**
Continue hovering the same link until the skeleton resolves.
Confirm: Tooltip shows the linked page's OG data — image (if present), title, description, and domain name.

**Test 3 — Dismiss (TTIP-02):**
Move the mouse away from the link.
Confirm: Tooltip disappears immediately.

**Test 4 — Error/empty state (TTIP-04):**
Find a link to a page without OG tags (e.g., a plain GitHub repo file, raw text URL, or PDF link).
Hover for 300ms until tooltip resolves.
Confirm: Tooltip shows "No preview available" or similar error message — NOT a blank white box.

**Test 5 — Viewport edge clamping:**
Hover a link in the bottom-right corner of the page.
Confirm: Tooltip stays within the visible viewport (does not get clipped or scroll the page).

**Test 6 — Shadow DOM isolation:**
Open DevTools → Elements tab → find `og-preview-tooltip` element in `<body>`.
Confirm: It has a shadow root. The tooltip's styling (card border, background, typography) looks correct regardless of the host page's styles.
