  <files>
    package.json

    lib/og-parser.ts
  </files>
  <action>
1. Install runtime dependencies:
   ```
   pnpm add htmlparser2 @webext-core/messaging
   ```

2. Create `lib/types.ts` with the `OgData` interface covering all fields from the research:
   - Core OG: title, description, url, siteName, type, locale
   - Image OG: image, imageAlt, imageWidth, imageHeight, imageType
   - Twitter Card: twitterCard, twitterSite, twitterCreator, twitterTitle, twitterDescription, twitterImage, twitterImageAlt
   - All fields optional strings (pages may have none, some, or all)

3. Create `lib/og-parser.ts` with these exports:

   a. `parseOgTags(html: string): Record<string, string>` -- uses htmlparser2 Parser with:
      - `onopentag` handler checking `name === 'meta'`, reading `property` or `name` attribute, keeping entries where key starts with `og:` or `twitter:`
      - `onclosetag` handler that sets a `done` boolean flag when `name === 'head'` (do NOT call `parser.pause()` -- uncertain in v10; instead check the flag in onopentag and return early)
      - Options: `{ decodeEntities: true, lowerCaseTags: true, lowerCaseAttributeNames: true }`
      - Call `parser.write(html)` then `parser.end()`
      - Return the collected `Record<string, string>`

   b. `normalizeOgData(raw: Record<string, string>): OgData` -- maps raw tag names to OgData fields exactly as shown in the research code example. Handle `og:image:url` as fallback for `og:image`, and `twitter:image:src` as fallback for `twitter:image`.

   c. `extractOgFromDOM(): Record<string, string>` -- uses `document.querySelectorAll` to read meta tags from the live DOM. Selectors: `meta[property^="og:"]`, `meta[property^="twitter:"]`, `meta[name^="twitter:"]`. For each element, read `property` or `name` attribute as key, `content` attribute as value. This function will be called by the content script (it uses DOM APIs, not htmlparser2).

   d. `getEffectiveTitle(data: OgData): string | undefined` -- returns `data.twitterTitle || data.title`
   e. `getEffectiveDescription(data: OgData): string | undefined` -- returns `data.twitterDescription || data.description`
   f. `getEffectiveImage(data: OgData): string | undefined` -- returns `data.twitterImage || data.image`
  </action>

Run `pnpm run typecheck` -- must pass with zero errors. Confirm htmlparser2 and @webext-core/messaging appear in package.json dependencies. Confirm lib/types.ts and lib/og-parser.ts exist and export the expected symbols.
  </verify>
  <done>
OgData interface defined. parseOgTags parses HTML strings via htmlparser2. normalizeOgData maps raw tags to typed fields. extractOgFromDOM reads from live DOM. All three effective-value helpers exported. TypeScript compiles cleanly.

<task type="auto">
  <name>Task 2: Create messaging protocol and session cache</name>
  <files>
    lib/messaging.ts
    lib/cache.ts
  </files>
  <action>
1. Create `lib/messaging.ts`:
   - Import `defineExtensionMessaging` from `@webext-core/messaging`
   - Import `OgData` from `./types`
   - Define `OgProtocolMap` interface with two message types:
     - `getOgData(data: { url: string }): OgData | null` -- service worker fetches+parses a remote URL and returns OG data (used by popup and tooltip in future phases)
     - `getPageOgData(data: { tabId: number }): OgData | null` -- service worker asks the content script on a specific tab to extract DOM meta tags and return them (used by popup to get current page OG data)
   - Export `sendMessage` and `onMessage` from `defineExtensionMessaging<OgProtocolMap>()`

   NOTE on @webext-core/messaging API: The protocol map defines message names as keys. The `data` parameter is the payload sent with the message. The return type is what the handler returns. Verify the exact API shape against the installed package -- if the generic parameter format differs, adapt accordingly. The key goal is type-safe messaging between content script and service worker.

2. Create `lib/cache.ts`:
   - Import `storage` from `wxt/storage` (WXT auto-provides this)
   - Import `OgData` from `./types`
   - Define `CACHE_PREFIX = 'session:ogcache:'` -- the `session:` prefix tells WXT storage to use `chrome.storage.session`
   - Export `getCachedOgData(url: string): Promise<OgData | null>` -- calls `storage.getItem<OgData>(\`${CACHE_PREFIX}${url}\`)`
   - Export `setCachedOgData(url: string, data: OgData): Promise<void>` -- calls `storage.setItem(\`${CACHE_PREFIX}${url}\`, data)`
   - Keep it simple: no eviction logic yet (10MB quota is generous for OgData objects at ~1-5KB each; eviction can be added later if needed)
  </action>

Run `pnpm run typecheck` -- must pass with zero errors. Confirm lib/messaging.ts exports sendMessage and onMessage. Confirm lib/cache.ts exports getCachedOgData and setCachedOgData.
  </verify>

Type-safe messaging protocol defined with getOgData and getPageOgData messages. Session cache provides get/set for OgData by URL. All four lib/ files compile cleanly together.
  </done>
