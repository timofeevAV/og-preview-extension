# Plan 08-04: Swiss Design + Theme System

## Goal
Apply Swiss Design principles (strict grid, type hierarchy, whitespace, achromatic palette, clean borders) to all popup components. Wire the `defaultTab` setting into `ExpandedView`. Implement tooltip theme override: when `theme` is `light` or `dark`, the shadow DOM tooltip should follow the forced theme rather than the system media query.

## must_haves
- All popup components use Swiss Design: 8px grid, system-ui font, borders not shadows, clean hierarchy
- `ExpandedView` reads `defaultTab` prop (from `App.tsx`) instead of hardcoded `'previews'`
- Tooltip CSS supports forced theme via `:host([data-theme="dark"])` and `:host([data-theme="light"])`
- Content script reads `theme` setting and sets `data-theme` attribute on the shadow host
- `popup/style.css` reduces `--radius` to `0.25rem` (4px) for Swiss crispness, or adjusts per component

### Tooltip theme override in shadow DOM

**In tooltip/style.css:**

Add forced theme overrides after the existing `@media (prefers-color-scheme: dark)` block:

```css
/* Forced light theme — overrides system dark preference */
:host([data-theme="light"]) {
  --background: oklch(1 0 0);
  --foreground: oklch(0.145 0 0);
  --card: oklch(1 0 0);
  --card-foreground: oklch(0.145 0 0);
  --muted: oklch(0.97 0 0);
  --muted-foreground: oklch(0.556 0 0);
  --primary: oklch(0.205 0 0);
  --border: oklch(0.922 0 0);
}

/* Forced dark theme — overrides system light preference */
:host([data-theme="dark"]) {
  --background: oklch(0.145 0 0);
  --foreground: oklch(0.985 0 0);
  --card: oklch(0.205 0 0);
  --card-foreground: oklch(0.985 0 0);
  --muted: oklch(0.269 0 0);
  --muted-foreground: oklch(0.708 0 0);
  --primary: oklch(0.87 0 0);
  --border: oklch(1 0 0 / 10%);
}
```

**In content.tsx:**

After creating the shadow UI (`ui.mount()`), read the `theme` setting and apply it to the shadow host:

```ts
const settings = await getSettings();
// Apply initial theme to shadow host
function applyShadowTheme(theme: OgPreviewSettings['theme']) {
  const host = document.querySelector('og-preview-tooltip');
  if (!host) return;
  if (theme === 'system') {
    host.removeAttribute('data-theme');
  } else {
    host.setAttribute('data-theme', theme);
  }
}
applyShadowTheme(settings.theme);

// Update theme attribute when setting changes
onSettingsChanged((changes) => {
  if ('theme' in changes && changes.theme !== undefined) {
    applyShadowTheme(changes.theme);
  }
  // ... existing hoverPreview/hoverDelay handling
});
```

Note: The shadow host element tag name is `og-preview-tooltip` (the `name` from `createShadowRootUi`). Use `document.querySelector('og-preview-tooltip')` to get it. Add this applyShadowTheme call after `ui.mount()`.
</task>
