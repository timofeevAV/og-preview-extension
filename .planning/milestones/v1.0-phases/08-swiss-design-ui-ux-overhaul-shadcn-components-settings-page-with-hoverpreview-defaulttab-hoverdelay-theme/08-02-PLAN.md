# Plan 08-02: Content Script Settings Integration

## Goal
Make `content.tsx` read `hoverPreview` and `hoverDelay` from `chrome.storage.sync` at init time. If `hoverPreview` is `false`, skip hover delegation. If it changes at runtime, enable/disable delegation without page reload. Make `hoverDelay` dynamic (currently hardcoded 300ms).

## must_haves
- Hover delegation is NOT set up if `hoverPreview: false` in storage
- Hover delegation IS set up if `hoverPreview: true`
- `hoverDelay` from storage is used as the timeout (not hardcoded 300)
- Changing `hoverPreview` or `hoverDelay` in settings takes effect without a page reload
- `setupHoverDelegation` returns a cleanup function (for teardown on setting change)

<task id="08-02-1">
### Refactor setupHoverDelegation to return cleanup

Modify `setupHoverDelegation` in `entrypoints/content.tsx`:

1. Add `delayMs: number` parameter (replaces hardcoded `300`)
2. Add `AbortController` to allow removing event listeners on teardown
3. Return `() => void` cleanup function

```ts
function setupHoverDelegation(controller: Controller, delayMs: number): () => void {
  const ac = new AbortController();
  const { signal } = ac;
  let timer: ReturnType<typeof setTimeout> | null = null;
  let activeUrl: string | null = null;

  document.addEventListener('mouseover', (e: MouseEvent) => {
    const link = (e.target as Element).closest('a[href]') as HTMLAnchorElement | null;
    if (!link) return;

    let url: string;
    try {
      const parsed = new URL(link.href);
      if (!['http:', 'https:'].includes(parsed.protocol)) return;
      url = parsed.href;
    } catch {
      return;
    }

    if (url === activeUrl) return;
    if (timer) { clearTimeout(timer); timer = null; }
    if (activeUrl) controller.hide();

    timer = setTimeout(() => {
      const rect = link.getBoundingClientRect();
      controller.show(url, rect.left, rect.top, rect.bottom);
      activeUrl = url;
      timer = null;
    }, delayMs);
  }, { signal });

  document.addEventListener('mouseout', (e: MouseEvent) => {
    const link = (e.target as Element).closest('a[href]') as HTMLAnchorElement | null;
    if (!link) return;
    if (link.contains(e.relatedTarget as Node)) return;
    if (timer) { clearTimeout(timer); timer = null; }
    controller.hide();
    activeUrl = null;
  }, { signal });

  return () => ac.abort();
}
```

<task id="08-02-2">
### Read settings at init and conditionally enable delegation

Replace the current `setupHoverDelegation(controller)` call in the `main` function with settings-aware init:

```ts
import { getSettings, onSettingsChanged } from '@/lib/settings';

// Inside main(ctx):
const settings = await getSettings();
let cleanup: (() => void) | null = null;

if (settings.hoverPreview) {
  cleanup = setupHoverDelegation(controller, settings.hoverDelay);
}

// Track current hoverDelay locally for storage change handler
let currentDelay = settings.hoverDelay;

onSettingsChanged((changes) => {
  if ('hoverDelay' in changes && changes.hoverDelay !== undefined) {
    currentDelay = changes.hoverDelay;
  }

  if ('hoverPreview' in changes) {
    if (changes.hoverPreview === false && cleanup) {
      cleanup();
      cleanup = null;
      controller.hide();
    } else if (changes.hoverPreview === true && !cleanup) {
      cleanup = setupHoverDelegation(controller, currentDelay);
    }
  } else if ('hoverDelay' in changes && cleanup) {
    // Restart delegation with new delay
    cleanup();
    cleanup = setupHoverDelegation(controller, currentDelay);
  }
});
```

The `onSettingsChanged` call does NOT need to be unsubscribed at `ctx.onInvalidated` since the content script is torn down by WXT anyway when the page unloads.
