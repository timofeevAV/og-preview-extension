# Roadmap: OG Preview Extension

## Overview

This roadmap delivers a Chromium MV3 browser extension that shows pixel-accurate social media card previews for any webpage. The journey starts with a working extension skeleton and OG data pipeline, builds the popup UI with platform previews and metadata tools, then tackles the highest-complexity feature -- hover-on-link OG tooltips -- as a standalone phase. Seven phases, each delivering a verifiable capability, covering all 22 v1 requirements.

## Phase Details

### Phase 1: Foundation
**Goal**: Extension can be installed and runs on Chrome, Edge, and Brave with correct MV3 manifest and permissions
**Depends on**: Nothing (first phase)
**Requirements**: CORE-03
**Success Criteria** (what must be TRUE):
  1. Extension installs and loads without errors on Chrome, Edge, and Brave
  2. Clicking the toolbar icon opens a popup (even if placeholder content)
  3. Manifest uses optional_host_permissions (not host_permissions) for broad URL access
  4. Development environment supports HMR for popup and content script changes
**Plans**: 1 plan

Plans:
- [ ] 01-01-PLAN.md -- Scaffold WXT project, configure manifest, create entrypoints, verify cross-browser install

### Phase 2: Data Pipeline
**Goal**: Extension can extract OG metadata from the current page and fetch/parse OG data from remote URLs
**Depends on**: Phase 1
**Requirements**: CORE-01, CORE-04
**Success Criteria** (what must be TRUE):
  1. Content script reads all og:* and twitter:* meta tags from the current page DOM and returns structured data
  2. Service worker accepts a URL, fetches it with timeout, parses OG tags using htmlparser2, and returns structured data
  3. Fetched OG data is cached in chrome.storage.session so repeat requests are instant
  4. OG parsing works correctly on localhost and local development URLs
**Plans**: 2 plans

Plans:
- [ ] 02-01-PLAN.md -- Install deps, create shared types, OG parser, messaging protocol, and session cache
- [ ] 02-02-PLAN.md -- Wire content script and service worker with fetch+parse pipeline and message handling

### Phase 7: Hover Tooltip
**Goal**: Users can hover over any link on a page to see an OG preview tooltip without leaving the page
**Depends on**: Phase 2
**Requirements**: TTIP-01, TTIP-02, TTIP-03, TTIP-04
**Success Criteria** (what must be TRUE):
  1. Hovering over a link for 300ms triggers a tooltip showing the linked page's OG preview
  2. Moving the mouse away from the link dismisses the tooltip
  3. While OG data is being fetched, the tooltip shows a loading skeleton
  4. When a linked page has no OG tags or the fetch fails, the tooltip shows a clear error/empty state
  5. Tooltip renders inside Shadow DOM and does not leak styles to or from the host page
