  <action>
1. **LICENSE** — Create MIT license file:
   - Use the standard MIT license text from choosealicense.com
   - Year: 2026
   - Copyright holder: "OG Preview Extension contributors" (generic to avoid personal name requirement)

2. **CONTRIBUTING.md** — Create contribution guidelines covering:
   - **Prerequisites:** Node.js 22+, pnpm 10+
   - **Getting started:** Clone, `pnpm install`, `pnpm dev` (Chrome), `pnpm dev:edge` (Edge)
   - **Available scripts:** `pnpm dev`, `pnpm build`, `pnpm typecheck`, `pnpm lint`, `pnpm lint:fix`, `pnpm format`, `pnpm format:check`, `pnpm test`
   - **Project structure:** Brief description of entrypoints (background, content, popup), lib/, components/
   - **Making changes:** Branch from `main`, make changes, run checks (`pnpm lint && pnpm format:check && pnpm typecheck && pnpm test`), submit PR
   - **Pre-commit hooks:** Note that `simple-git-hooks` + `lint-staged` auto-run lint and format on staged files
   - **Code style:** ESLint (strictTypeChecked) + Prettier enforced; no manual configuration needed
   - **PR guidelines:** Descriptive title, reference related issues, ensure CI passes
   - Keep it concise — no more than ~100 lines. This is a small project; don't over-document.

3. **CODE_OF_CONDUCT.md** — Use Contributor Covenant v2.1 (the industry standard):
   - Full text from https://www.contributor-covenant.org/version/2/1/code_of_conduct/
   - Contact method: GitHub Issues (or repository owner's public profile)

4. **SECURITY.md** — Create security policy:
   - State that security issues should NOT be filed as public issues
   - Provide instructions to report via GitHub's private security advisory feature (Settings > Security > Advisories > "Report a vulnerability")
   - State the project's response timeline expectation (e.g., acknowledge within 48 hours)
   - Keep brief — ~20 lines
  </action>
