<task type="auto">
  <name>Task 1: Create GitHub Actions CI workflow, .node-version, .editorconfig, and add packageManager to package.json</name>
  <files>.github/workflows/ci.yml, .node-version, .editorconfig, package.json</files>
  <action>
1. Create `.github/workflows/ci.yml` with a single CI workflow:
   - Triggers: `push` to `main` branch, `pull_request` to `main` branch
   - Concurrency group: `${{ github.workflow }}-${{ github.event.pull_request.number || github.sha }}` with `cancel-in-progress: ${{ github.event_name == 'pull_request' }}`
   - Single job `ci` named "Lint, Test & Build" running on `ubuntu-latest`
   - Steps in order:
     a. `actions/checkout@v4`
     b. `pnpm/action-setup@v4` (no version — reads from packageManager field)
     c. `actions/setup-node@v4` with `node-version-file: '.node-version'` and `cache: 'pnpm'`
     d. `pnpm install --frozen-lockfile`
     e. `pnpm lint`
     f. `pnpm format:check`
     g. `pnpm typecheck`
     h. `pnpm test`
     i. `pnpm build`
     j. `actions/upload-artifact@v4` (conditional: `if: github.ref == 'refs/heads/main'`) with name `og-preview-chrome-mv3`, path `.output/chrome-mv3/`, retention 30 days

2. Create `.node-version` containing just `22` (Node.js 22 LTS — maintained until April 2027). The project runs on Node 25 locally but CI should target the stable LTS version.

3. Create `.editorconfig` with:
   - `root = true`
   - `[*]`: charset utf-8, end_of_line lf, indent_style space, indent_size 2, insert_final_newline true, trim_trailing_whitespace true
   - `[*.md]`: trim_trailing_whitespace false

4. Add `"packageManager": "pnpm@10.26.1"` to package.json (matching local pnpm version). Add this field at the top level, after the `"type"` field. This enables `pnpm/action-setup@v4` to auto-detect the correct pnpm version and enables corepack support for contributors.

Do NOT:
- Create matrix builds for multiple Node versions or browsers (this is a single Chrome MV3 extension)
- Create separate workflows per check (lint.yml, test.yml, etc.) -- one workflow is correct for this project size
- Use `actions/cache` directly -- `setup-node` handles caching via `cache: 'pnpm'`
  </action>

- `node -e "const p = require('./package.json'); console.log(p.packageManager)"` outputs `pnpm@10.26.1`
- `pnpm lint && pnpm format:check && pnpm typecheck && pnpm test && pnpm build` all still pass (nothing broken)
