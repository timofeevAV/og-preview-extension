<task type="auto">
  <name>Task 1: Update .gitignore and configure GitHub repository via gh CLI</name>
  <files>.gitignore</files>
  <action>
1. **Update `.gitignore`** — Add these entries to exclude internal planning files from the public repository:
   ```
   # Internal planning (GSD workflow)
   .planning/
   ```
   Note: Do NOT add CLAUDE.md to .gitignore — it contains useful project context for AI-assisted development and should remain in the public repo. The research flagged this as a user decision; keeping it is the reasonable default since it provides value to contributors using Claude Code.

   Append the new entries to the existing .gitignore (do not overwrite existing entries for .wxt/, .output/, node_modules/, web-ext.config.ts, .DS_Store, Thumbs.db).

2. **Configure repository metadata via `gh repo edit`:**
   ```bash
   gh repo edit --description "Chrome extension to preview Open Graph cards for any webpage or link" \
     --add-topic browser-extension \
     --add-topic chrome-extension \
     --add-topic open-graph \
     --add-topic og-tags \
     --add-topic social-media-preview \
     --add-topic wxt \
     --add-topic react \
     --add-topic typescript \
     --enable-discussions=false \
     --enable-wiki=false \
     --delete-branch-on-merge
   ```

3. **Create issue labels via `gh label create`** (use `-f` flag to force-update if label already exists):
   ```bash
   gh label create "bug" -d "Something isn't working" -c "d73a4a" -f
   gh label create "enhancement" -d "New feature or request" -c "a2eeef" -f
   gh label create "good first issue" -d "Good for newcomers" -c "7057ff" -f
   gh label create "documentation" -d "Improvements or additions to documentation" -c "0075ca" -f
   gh label create "ci" -d "CI/CD and automation" -c "e4e669" -f
   gh label create "dependencies" -d "Pull requests that update a dependency" -c "0366d6" -f
   ```

4. **Rename default branch from `master` to `main`:**
   ```bash
   gh repo rename-branch master main
   ```
   This aligns with GitHub's modern convention. The CI workflow references `main` branch. After rename, update local tracking:
   ```bash
   git branch -m master main
   git fetch origin
   git branch -u origin/main main
   ```

If the `gh` CLI is not authenticated or the repo doesn't exist on GitHub, these commands will fail. That is acceptable — the executor should create a checkpoint for the user to authenticate. The `.gitignore` update is the critical automatable piece.
